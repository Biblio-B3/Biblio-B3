# Biblio-B3

Biblio-B3 est une application web complète de gestion de bibliothèque. Elle permet de gérer les livres, les exemplaires, les utilisateurs, les réservations et les avis. L'application est composée d'un backend en Node.js avec Express et d'un frontend en Next.js.

## Aperçu du projet

L'application Biblio-B3 est conçue pour être une solution complète pour la gestion d'une bibliothèque. Elle offre une interface pour les administrateurs pour gérer le catalogue de livres, les exemplaires physiques, les utilisateurs et suivre les statistiques d'utilisation. Les utilisateurs peuvent parcourir les livres, les réserver, et laisser des avis.

## Stack technique

### Backend

-   **Node.js**: Environnement d'exécution JavaScript.
-   **Express**: Framework web pour Node.js.
-   **TypeScript**: Superset de JavaScript qui ajoute des types statiques.
-   **PostgreSQL**: Système de gestion de base de données relationnelle.
-   **Drizzle ORM**: ORM TypeScript pour interagir avec la base de données.
-   **JWT (JSON Web Tokens)**: Pour l'authentification sécurisée.
-   **Swagger**: Pour la documentation de l'API.

### Frontend

-   **Next.js**: Framework React pour la production.
-   **React**: Bibliothèque JavaScript pour la construction d'interfaces utilisateur.
-   **TypeScript**: Pour un développement plus robuste.
-   **Tailwind CSS**: Framework CSS pour un design rapide et personnalisé.
-   **Shadcn/ui**: Composants d'interface utilisateur réutilisables.

## Fonctionnalités

-   **Gestion des livres**: Ajouter, modifier, supprimer et archiver des livres. Importation de livres via ISBN avec récupération des informations depuis Google Books.
-   **Gestion des exemplaires**: Gérer les exemplaires physiques de chaque livre, y compris leur état.
-   **Gestion des utilisateurs**: Inscription, connexion, et gestion des profils utilisateurs.
-   **Système de réservation**: Les utilisateurs peuvent réserver des livres disponibles.
-   **Gestion des avis**: Les utilisateurs peuvent laisser des notes et des commentaires sur les livres.
-   **Statistiques**: Suivi des livres les plus populaires, des utilisateurs les plus actifs, et d'autres statistiques pertinentes.
-   **Authentification JWT**: Système d'authentification sécurisé basé sur les tokens.

## Démarrage rapide

Pour lancer l'application en environnement de développement local :

1.  **Clonez le dépôt**:
    ```bash
    git clone https://github.com/votre-utilisateur/Biblio-B3.git
    cd Biblio-B3
    ```

2.  **Configurez les variables d'environnement**:
    Créez des fichiers `.env` dans les répertoires `backend` et `frontend` en vous basant sur les exemples `.env.docker`.

3.  **Lancez les services avec Docker Compose**:
    ```bash
    docker-compose -f compose.yml up -d
    ```
    Cette commande démarrera la base de données PostgreSQL et pgAdmin.

4.  **Installez les dépendances et lancez le backend**:
    ```bash
    cd backend
    yarn install
    yarn dev
    ```

5.  **Installez les dépendances et lancez le frontend**:
    ```bash
    cd ../frontend
    yarn install
    yarn dev
    ```

L'application frontend sera accessible à `http://localhost:4000` et le backend à `http://localhost:3001`.

## Déploiement

### Avec Docker Compose

Pour le déploiement en production, un fichier `compose.yml` est fourni dans le répertoire `deployment`.

1.  **Placez-vous dans le répertoire de déploiement**:
    ```bash
    cd deployment
    ```

2.  **Configurez les variables d'environnement**:
    Assurez-vous que les fichiers `.env.docker` dans les répertoires `backend` et `frontend` sont correctement configurés pour la production.

3.  **Lancez les services**:
    ```bash
    docker-compose up -d
    ```
    Cette commande déploiera le backend, le frontend, la base de données, et un serveur Nginx comme reverse proxy.

## Documentation de l'API

### Authentification

L'API utilise des JSON Web Tokens (JWT) pour l'authentification. Un token est généré lors de la connexion et doit être inclus dans l'en-tête `Authorization` de chaque requête sécurisée.

**Exemple d'en-tête**: `Authorization: Bearer <votre_token_jwt>`

### Endpoints de l'API

Voici un aperçu des principaux endpoints de l'API. Pour une documentation complète, une fois le backend lancé, vous pouvez consulter l'interface Swagger à l'adresse `http://localhost:3001/api-docs`.

-   `POST /login`: Connexion d'un utilisateur.
-   `POST /registration`: Inscription d'un nouvel utilisateur.
-   `POST /logout/:id`: Déconnexion d'un utilisateur.
-   `POST /reset-password`: Demande de réinitialisation de mot de passe.
-   `POST /reset-password/confirm`: Confirmation de la réinitialisation de mot de passe.

-   `GET /books`: Récupérer la liste des livres.
-   `GET /books/:id`: Récupérer les détails d'un livre.
-   `POST /books/import`: Importer un livre via son ISBN.
-   `POST /books/manual`: Ajouter un livre manuellement.
-   `PUT /books/:id`: Mettre à jour un livre.
-   `DELETE /books/:id`: Supprimer un livre.

-   `GET /copy`: Récupérer la liste des exemplaires.
-   `POST /copy`: Ajouter un exemplaire.
-   `PUT /copy/:id`: Mettre à jour un exemplaire.
-   `DELETE /copy/:id`: Supprimer un exemplaire.

-   `GET /reservations`: Récupérer la liste des réservations.
-   `POST /reservations`: Créer une nouvelle réservation.
-   `DELETE /reservations/:id`: Annuler une réservation.

-   `GET /reviews`: Récupérer la liste des avis.
-   `POST /reviews`: Ajouter un avis.
-   `PUT /reviews/:id`: Mettre à jour un avis.
-   `DELETE /reviews/:id`: Supprimer un avis.

-   `GET /stats/...`: Endpoints pour les statistiques.
